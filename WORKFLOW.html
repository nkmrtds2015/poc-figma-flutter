Figma × Claude Code × Flutter ワークフロー手順書

作成日: 2026-05-17  |  環境: Windows 11 + WSL2 (Ubuntu 24.04) + Flutter 3.41.9 + Claude Code (VSCode Remote WSL)
GitHub: github.com/nkmrtds2015/poc-figma-flutter

目次 1. ワークフロー概要 2. 環境セットアップ（初回のみ） 3. Figma → Flutter コード生成手順 4. Design Tokens 双方向同期手順 5. ビルド・実行手順 6. 自動化できるステップ・手動が残るステップ 7. 本番（組み込み Linux）への移行課題

1. ワークフロー概要

【Figma → Flutter（UI コード生成）】
  Figma デザインファイル
      ↓  Figma MCP (figma-developer-mcp)
  Claude Code がデザイン構造・色・座標を取得
      ↓  コード自動生成
  Flutter ウィジェットコード (lib/screens/, lib/widgets/, lib/theme/)
      ↓  flutter build linux
  Linux デスクトップアプリ (WSLg → Windows ウィンドウとして表示)

【Figma ↔ Flutter（Design Tokens 双方向）】
  Figma Variables (色・スペース・フォント)
      ↕  tokens.json (W3C Design Tokens 形式)
  Flutter AppTheme / const 定義 (Dart コード)
  ※ 逆方向 (Flutter → Figma) は Token Studio プラグインで手動 import

各ツールの役割

ツール役割動作場所
Figma (ブラウザ/デスクトップ)デザイン作成・Variables 管理Windows 側
figma-developer-mcpFigma API ↔ Claude Code の橋渡しWSL2 (Node.js)
Claude Code (VSCode Remote WSL)デザイン解析・コード生成・レビューWSL2
Flutter SDK 3.41.9ビルド・実行WSL2
WSLgLinux GUI を Windows ウィンドウで表示Windows 11 組み込み

2. 環境セットアップ（初回のみ）

Step 1: GTK 依存パッケージのインストール
sudo apt update
sudo apt install -y clang cmake ninja-build pkg-config libgtk-3-dev liblzma-dev libstdc++-12-dev
Step 2: Flutter SDK のインストール
# 最新安定版バージョンを確認
python3 -c "
import json, urllib.request
d = json.load(urllib.request.urlopen('https://storage.googleapis.com/flutter_infra_release/releases/releases_linux.json'))
r = [x for x in d['releases'] if x['channel']=='stable'][0]
print(r['version'], r['archive'])
"

# ダウンロード・展開（バージョンは確認した値に置き換える）
cd ~
wget https://storage.googleapis.com/flutter_infra_release/releases/stable/linux/flutter_linux_3.41.9-stable.tar.xz
tar xf flutter_linux_3.41.9-stable.tar.xz
echo 'export PATH="$HOME/flutter/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc

flutter config --enable-linux-desktop
flutter doctor  # Linux toolchain: ✓ が表示されれば OK
Step 3: Figma アカウント作成 & Personal Access Token 発行
  1. Windows のブラウザで figma.com にアクセスし無料アカウント作成
  2. 右上アイコン → Settings → Account → Personal access tokens
  3. 「Generate new token」で発行
    • Name: claude-code-poc
    • Expiration: 30 days
    • Scopes: File content: Read only のみ選択 ⚠️ Variables スコープは無料プランでは表示されない（Enterprise 限定）
  4. 発行されたトークンを WSL2 ターミナルで安全保存: 
    echo 'FIGMA_ACCESS_TOKEN=<発行されたトークン>' > ~/.figma_token
chmod 600 ~/.figma_token
Step 4: Figma MCP の登録（VSCode Remote WSL 環境）
⚠️ 重要: VSCode Remote (WSL) 拡張経由で Claude Code を使う場合、 claude mcp add コマンドは Windows 側の C:\Users\...\.claude.json に書き込む。 しかし実行環境は WSL2 側のため、WSL 側の ~/.claude.json に登録する必要がある。 VSCode Server 内の Claude バイナリを使って登録すること。 
# vscode-server 内の Claude バイナリのパスを確認
CLAUDE_BIN=$(ls ~/.vscode-server/extensions/anthropic.claude-code-*/resources/native-binary/claude 2>/dev/null | tail -1)
echo $CLAUDE_BIN  # パスが表示されれば OK

# トークンを読み込んで MCP 登録
TOKEN=$(grep FIGMA_ACCESS_TOKEN ~/.figma_token | cut -d= -f2- | tr -d '\n')
$CLAUDE_BIN mcp add figma -s user -- npx -y figma-developer-mcp --figma-api-key $TOKEN --stdio

# 登録確認（figma: ✓ Connected が表示されれば OK）
$CLAUDE_BIN mcp list

登録後、VSCode を完全に終了して再起動する（Reload Window では MCP が反映されない）。

Step 5: WSLg の動作確認
echo $DISPLAY        # ":0" が返れば WSLg 有効
echo $WAYLAND_DISPLAY  # "wayland-0" が返れば Wayland も有効

3. Figma → Flutter コード生成手順

Step 1: Figma でデザインを作成
  • Windows のブラウザまたは Figma デスクトップアプリでデザインを作成
  • デザインファイルの URL をコピー (https://www.figma.com/design/<fileKey>/...)
    ⚠️ figma.com/make/... (Figma Make) の URL は MCP で読めない。通常の Figma デザインとして開き直すこと。
Step 2: Claude Code で MCP ツールを使ってデザインデータを取得

Claude Code のチャットで以下を指示する:

「Figma MCP で https://www.figma.com/design/<fileKey>/... のデザインデータを取得して、Flutter コードを生成してください」

Claude Code が内部で mcp__figma__get_figma_data ツールを呼び出し、以下の情報を取得する:

  • ノード構造（Frame・Rectangle・Text など）
  • 座標・サイズ・色・フォントスタイル
  • コンポーネント定義
Step 3: 生成されるファイル構成
lib/
├── main.dart                        # エントリーポイント
├── screens/
│   └── dashboard_screen.dart        # Figma Frame から生成された画面
├── widgets/
│   └── card_widget.dart             # Figma Component から生成されたウィジェット
└── theme/
    └── app_theme.dart               # Figma の色・テキストスタイルを定数として定義
Step 4: コード品質確認
export PATH="$HOME/flutter/bin:$PATH"
cd poc_app
flutter analyze  # "No issues found!" が返れば OK

4. Design Tokens 双方向同期手順

4-1. Figma → Flutter（順方向・自動）

Step 1: Figma で Token Studio プラグインをインストール
  1. Figma → Plugins → 「Token Studio for Figma」を検索してインストール
  2. Figma Variables を Token Studio で管理する形に設定
Step 2: tokens.json を Export
  1. Token Studio プラグインを開く
  2. Export → JSON として tokens.json を保存
  3. ファイルを WSL2 プロジェクトフォルダにコピー
Step 3: tokens_to_dart.py で Dart コードに変換（実装済み）
✅ 実装済み: figma2flutter パッケージは使わず、自前の Python スクリプトで同等機能を実装。双方向同期に対応している点が自前ツールの強み。
cd poc_app
python3 tools/tokens_to_dart.py
# → lib/theme/app_theme.dart が自動生成される

4-2. Flutter → Figma（逆方向）

⚠️ 無料プラン制約: Figma REST API の Variables 書き込み (file_variables:write) は Enterprise プランのみ。色以外（テキスト・サイズ）は REST API では書き込み不可。 現行実装では 色のみ自動・テキスト/サイズは手動 で Figma を編集する。
Step 1: app_theme.dart の変更を tokens.json に書き出す（色のみ自動）
python3 tools/dart_to_tokens.py
# app_theme.dart の AppColors クラスから色定数を抽出し tokens.json を更新
# 変更があれば差分を表示して次のステップを案内する
Step 2: Figma の Token Studio で colors を import（色）
  1. Windows 側の Figma を開く
  2. Token Studio プラグイン → Import JSON → tokens.json を選択
  3. Figma Variables（色）が更新されたことを確認
Step 3: テキスト・サイズの変更は Figma を直接手動編集
  • テキスト変更: 該当レイヤーをダブルクリックして編集
  • サイズ・座標変更: 右パネルで数値を修正
  • 将来的な自動化: figma-use (Chrome DevTools Protocol) を使えばテキスト・サイズも自動反映可能（Phase 6・保留中）

5. ビルド・実行手順

デバッグ実行（WSLg 経由で Windows ウィンドウとして表示）
export PATH="$HOME/flutter/bin:$PATH"
export DISPLAY=:0
cd poc_app
flutter run -d linux
リリースビルド

# 成果物: build/linux/x64/release/bundle/poc_app

6. 自動化できるステップ・手動が残るステップ

ステップ自動/手動備考
Figma デザインデータの取得✅ 自動Figma MCP + Claude Code
Flutter UI コード生成✅ 自動Claude Code が生成
flutter analyze / build✅ 自動CI/CD に組み込み可
Figma Variables → tokens.json Export⚠️ 手動Token Studio プラグインで手動操作（無料プラン）
tokens.json → Dart コード変換✅ 自動python3 tools/tokens_to_dart.py（自前スクリプト）
Flutter 変更（色）→ tokens.json 更新✅ 自動python3 tools/dart_to_tokens.py（色のみ対応）
Flutter 変更（色）→ Figma Variables 反映⚠️ 手動Token Studio で手動 import（無料プラン制約）
Flutter 変更（テキスト・サイズ）→ Figma 反映⚠️ 手動Figma を直接手動編集（REST API 非対応のため）
GitHub へのコード登録✅ 自動git push

自動化の拡張余地

  • GitHub Actions で flutter build linux を CI に組み込み可能
  • Figma Webhook + tokens.json 自動生成スクリプトで Export を半自動化できる
  • Enterprise プランに移行すれば Variables の書き込みも API 自動化できる
  • figma-use を使えばテキスト・サイズの逆反映も自動化可能（下記補足参照）

7. 本番（組み込み Linux）への移行課題

課題内容優先度
ターゲット CPU アーキテクチャの確定 ARM (Raspberry Pi 等) か x86_64 か未確定。ARM の場合はクロスコンパイル設定が必要。 🔴 高
GTK 依存ライブラリのデプロイ Flutter Linux ビルドは libflutter_linux_gtk.so など動的ライブラリが必要。組み込み OS に GTK が入っているか確認が必要。 🔴 高
ARM 向けクロスコンパイル設定 Flutter の ARM Linux ビルドは現時点で公式サポートが限定的。flutter build linux --target-platform linux-arm64 の検証が必要。 🟡 中
ディスプレイサーバーの確認 組み込み環境で X11/Wayland が使えるか確認。使えない場合は Flutter の Embedded Linux (eLinux) 対応を検討。 🟡 中
タッチスクリーン対応 組み込み機器がタッチパネルを持つ場合、Flutter のタッチイベント対応を確認・実装。 🟢 低（要件次第）
フォント埋め込み 組み込み環境に Inter フォントが入っていない場合、pubspec.yaml にフォントをバンドルする設定が必要。 🟢 低

補足A: figma-use CLI について（Phase 6 候補技術）

GitHub: dannote/figma-use
Chrome DevTools Protocol (CDP) 経由で Figma Desktop を操作する OSS CLI ツール。 REST API では不可能なテキスト・サイズ・色の書き込みをターミナルから実行できる。

仕組み

WSL2 ターミナル
    ↓ npx figma-use <command>
Chrome DevTools Protocol (port 9222)
    ↓
Windows 側 Figma Desktop (Electron/Chromium)
    ↓ Plugin API 相当の操作
Figma キャンバス上のノード

主なコマンド

操作コマンド例
テキスト変更figma-use set-text --node-id 1:1684 --text "新テキスト"
色変更figma-use set-fill --node-id 1:1682 --color "#2196F3"
サイズ変更figma-use resize --node-id 1:1684 --width 580 --height 118
位置変更figma-use move --node-id 1:1684 --x 117 --y 209
ノード情報取得figma-use get --node-id 1:1684

前提条件

⚠️ Figma Desktop を特殊起動する必要がある: 
# Windows PowerShell で実行
& "C:\Users\<name>\AppData\Local\Figma\Figma.exe" --remote-debugging-port=9222
通常起動では CDP が無効なため、毎回このオプション付きで起動する必要がある。

制約・注意点

項目内容
Figma Desktop 必須ブラウザ版 Figma では動作しない
常時起動が必要操作中は Figma Desktop が開いていること
公式サポート外Figma の内部実装変更で突然動かなくなるリスクあり
ノード ID が必要事前に Figma MCP で各ノードの ID を取得しておくこと
アルファ品質2024年末リリースで実績がまだ少ない
CI/CD 不向きFigma Desktop の常時起動が必要なため、ローカル環境での利用が現実的

このプロジェクトでの位置づけ

Phase 6（保留中）の中核技術。現在「手動」になっているテキスト・サイズの Figma 逆反映を自動化するために使用予定。 実装する場合は tools/push_to_figma.sh から呼び出す形になる。

補足B: figma2flutter パッケージについて

pub.dev: figma2flutter
Design Tokens JSON → Flutter Dart コードに変換する Dart パッケージ。 本 PoC では使用していない（自前の tokens_to_dart.py で同等機能を実装済み）。 将来の移行検討のために調査結果を記録する。

評価軸figma2flutter自前ツール (tokens_to_dart.py)
色✅ グラデーション・修飾子も対応✅ 対応
タイポグラフィ✅ 全属性対応⚠️ title スタイルのみ
スペーシング✅ EdgeInsets として生成⚠️ 数値定数のみ（未リンク）
ボーダー・シャドウ✅ 対応❌ 未対応
複数テーマ (Dark Mode等)✅ ITokens インターフェース❌ 非対応
双方向同期 (Dart→Figma)❌ 一方向のみ✅ dart_to_tokens.py で対応
カスタマイズ⚠️ Dart 知識が必要✅ Python で容易に拡張可
メンテナンス状況⚠️ v0.3.1-alpha（不安定）⚠️ PoC 段階

使い分けの目安:

  • PoC・プロトタイプ規模 → 自前ツールで十分。双方向同期の強みあり。
  • 本番・大規模デザインシステム・複数テーマ → figma2flutter を再評価。ただし alpha 版のため安定性に注意。
